El backend del Sistema Aduanero Integrado está desarrollado en Java con Spring Boot. Aquí es donde vive toda la lógica del sistema: la comunicación con la base de datos, la seguridad con tokens JWT y la API REST que usan los módulos de Aduana, PDI y SAG. Para poder ejecutarlo se necesita tener instalado JDK 17, Maven y MySQL 8 o superior.

Antes de iniciar el servidor, debes crear la base de datos vacía llamada sistema_aduanero_db, ya que el proyecto genera automáticamente las tablas usando Hibernate. Después, tienes que revisar el archivo application.properties, ubicado en src/main/resources/. Ahí debes colocar tus credenciales de MySQL para que la aplicación pueda conectarse correctamente. También encontrarás los parámetros relacionados con el JWT: la clave secreta y el tiempo de expiración del token (por defecto, 24 horas). Otro punto importante es la configuración CORS, que por defecto solo acepta peticiones desde http://localhost:5173. Si el frontend está en otro puerto, tendrás que actualizar esa parte del código.

Para iniciar el backend, solo debes ejecutar mvn spring-boot:run desde la raíz del proyecto. Cuando el servidor esté funcionando, lo encontrarás en el puerto 8080.

La estructura del sistema está dividida por responsabilidades claras. AuthController maneja el registro y la autenticación. DashboardController entrega estadísticas usando conteos directos sobre las tablas. ReportesController permite generar “snapshots” de los datos actuales, creando historiales estáticos. Finalmente, los controladores específicos (Aduana, PDI, SAG, Trámites, Documentos, etc.) se encargan del CRUD de cada entidad operativa.
